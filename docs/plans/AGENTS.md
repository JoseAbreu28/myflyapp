# Plans

## Purpose

- Stores dated feature and deployment plans used to guide past or future implementation.
- Existing plans cover the original PPL dashboard, SPA/Windy/LPPR conversion, and Vercel deployment.

## Ownership

- Owns `*.md` planning files in this folder.
- Current implementation contracts belong in the nearest active `AGENTS.md`, not only in plans.

## Local Contracts

- Preserve front matter fields when editing existing plans: `title`, `type`, `status`, `date`, and any `origin`.
- Keep plans scoped, testable, and explicit about requirements, boundaries, decisions, risks, and verification.
- Mark superseded assumptions clearly if editing old plans; do not silently rewrite history into current-state docs.

## Work Guidance

- Use filenames like `YYYY-MM-DD-NNN-short-slug-plan.md`.
- Write plans as implementation guidance, not as code to paste blindly.

## Verification

- Review Markdown rendering and internal links after edits.
