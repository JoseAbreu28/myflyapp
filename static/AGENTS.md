# Static Assets

## Purpose

- Contains browser-side CSS and JavaScript for the MyFlyApp SPA.
- Assets support METAR/TAF rendering, maps, route planning and simulation, manual instrument study, PDF export, i18n, and mass/balance calculations.

## Ownership

- Owns `css/` and `js/` as static browser assets.
- HTML structure is owned by `templates/`; API payload shape is owned by root `app.py`.

## Local Contracts

- Keep assets framework-free and compatible with the current Flask/Jinja templates.
- External browser libraries currently load from CDNs in `templates/base.html`; Leaflet-dependent code must guard against missing `window.L`.
- Preserve paths generated by Flask static URLs, including `/static/css/style.css`, `/static/js/app.js`, `/static/js/navigation.js`, `/static/js/hsi.js`, `/static/js/rmi.js`, `/static/js/vor-indicator.js`, `/static/js/instrument-lab.js`, and `/static/js/massbalance.js`.

## Work Guidance

- Coordinate selector, ID, and event changes with `templates/index.html`.
- Keep aviation calculations auditable; document source assumptions near the data or logic they affect.

## Verification

- Load `/static/css/style.css` and key JS files from a running local server.
- Browser-check the SPA after changing any shared asset.

## Child DOX Index

- `css/AGENTS.md` covers styling and print layout.
- `js/AGENTS.md` covers browser behavior and calculations.
